854 /******************************************************************************
855 * *
856 * :: Transformation functions *
857 * *
858 * TinySpline is a library focusing on transformations. That is, most *
859 * functions are used to transform splines by modifying their state, e.g., *
860 * their number of control points, their degree, and so on. Accordingly, each *
861 * transformation functions specifies an input and output parameter (along *
862 * with the other parameters required to calculate the actual transformation). *
863 * By passing a different pointer to the output parameter, the transformation *
864 * result is calculated and stored without changing the state of the input. *
865 * This is in particular useful when dealing with errors as the original state *
866 * will never be modified. For instance, let's have a look at the following *
867 * code snippet: *
868 * *
869 * tsBSpline in = ... // an arbitrary spline *
870 * tsBSpline out; // result of transformation *
871 * *
872 * // Subdivide 'in' into sequence of bezier curves and store the result *
873 * // in 'out'. Does not change 'in' in any way. *
874 * tsError err = ts_bspline_to_beziers(&in, &out); *
875 * if (err != TS_SUCCESS) { *
876 * // fortunately, 'in' has not been changed *
877 * } *
878 * *
879 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
880 * changed allowing you to handle the error properly. *
881 * *
882 * Unless stated otherwise, the order of the parameters for transformation *
883 * functions is: *
884 * *
885 * function(input, [additional_input], output, [additional_output]) *
886 * *
887 * 'additional_input' are parameters required to calculate the actual *
888 * transformation. 'additional_output' are parameters storing further result. *
889 * *
890 * Note: None of TinySpline's transformation functions frees the memory of the *
891 * output parameter. Thus, when using the same output parameter multiple *
892 * times, make sure to free memory before each call. Otherwise, you will *
893 * have a bad time with memory leaks: *
894 * *
895 * tsBSpline in = ... // an arbitrary spline *
896 * tsBSpline out; // result of transformations *
897 * *
898 * ts_bspline_to_beziers(&in, &out); // first transformation *
899 * ... // some code *
900 * ts_bspline_free(&out); // avoid memory leak. *
901 * ts_bspline_buckle(&in, &out); // next transformation *
902 * *
903 * If you want to modify your input directly without having a separate output, *
904 * pass it as input and output at once: *
905 * *
906 * tsBSpline s = ... // an arbitrary spline *
907 * tsReal *knots = ... // a knot vector *
908 * *
909 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
910 * *
911 * Note: If a transformation function fails *and* input != output, all fields *
912 * of the output parameter are set to 0/NULL. If input == output, your *
913 * input may have an invalid state in case of errors. *
914 * *
915 ******************************************************************************/
