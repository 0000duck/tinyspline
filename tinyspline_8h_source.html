874 /******************************************************************************
875 * *
876 * :: Transformation functions *
877 * *
878 * TinySpline is a library focusing on transformations. That is, most *
879 * functions are used to transform splines by modifying their state, e.g., *
880 * their number of control points, their degree, and so on. Accordingly, each *
881 * transformation functions specifies an input and output parameter (along *
882 * with the other parameters required to calculate the actual transformation). *
883 * By passing a different pointer to the output parameter, the transformation *
884 * result is calculated and stored without changing the state of the input. *
885 * This is in particular useful when dealing with errors as the original state *
886 * will never be modified. For instance, let's have a look at the following *
887 * code snippet: *
888 * *
889 * tsBSpline in = ... // an arbitrary spline *
890 * tsBSpline out; // result of transformation *
891 * *
892 * // Subdivide 'in' into sequence of bezier curves and store the result *
893 * // in 'out'. Does not change 'in' in any way. *
894 * tsError err = ts_bspline_to_beziers(&in, &out); *
895 * if (err != TS_SUCCESS) { *
896 * // fortunately, 'in' has not been changed *
897 * } *
898 * *
899 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
900 * changed allowing you to handle the error properly. *
901 * *
902 * Unless stated otherwise, the order of the parameters for transformation *
903 * functions is: *
904 * *
905 * function(input, [additional_input], output, [additional_output]) *
906 * *
907 * 'additional_input' are parameters required to calculate the actual *
908 * transformation. 'additional_output' are parameters storing further result. *
909 * *
910 * Note: None of TinySpline's transformation functions frees the memory of the *
911 * output parameter. Thus, when using the same output parameter multiple *
912 * times, make sure to free memory before each call. Otherwise, you will *
913 * have a bad time with memory leaks: *
914 * *
915 * tsBSpline in = ... // an arbitrary spline *
916 * tsBSpline out; // result of transformations *
917 * *
918 * ts_bspline_to_beziers(&in, &out); // first transformation *
919 * ... // some code *
920 * ts_bspline_free(&out); // avoid memory leak. *
921 * ts_bspline_buckle(&in, &out); // next transformation *
922 * *
923 * If you want to modify your input directly without having a separate output, *
924 * pass it as input and output at once: *
925 * *
926 * tsBSpline s = ... // an arbitrary spline *
927 * tsReal *knots = ... // a knot vector *
928 * *
929 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
930 * *
931 * Note: If a transformation function fails *and* input != output, all fields *
932 * of the output parameter are set to 0/NULL. If input == output, your *
933 * input may have an invalid state in case of errors. *
934 * *
935 ******************************************************************************/
