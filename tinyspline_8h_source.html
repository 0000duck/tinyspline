808 /******************************************************************************
809 * *
810 * :: Transformation functions *
811 * *
812 * TinySpline is a library focusing on transformations. That is, most *
813 * functions are used to transform splines by modifying their state, e.g., *
814 * their number of control points, their degree, and so on. Accordingly, each *
815 * transformation functions specifies an input and output parameter (along *
816 * with the other parameters required to calculate the actual transformation). *
817 * By passing a different pointer to the output parameter, the transformation *
818 * result is calculated and stored without changing the state of the input. *
819 * This is in particular useful when dealing with errors as the original state *
820 * will never be modified. For instance, let's have a look at the following *
821 * code snippet: *
822 * *
823 * tsBSpline in = ... // an arbitrary spline *
824 * tsBSpline out; // result of transformation *
825 * *
826 * // Subdivide 'in' into sequence of bezier curves and store the result *
827 * // in 'out'. Does not change 'in' in any way. *
828 * tsError err = ts_bspline_to_beziers(&in, &out); *
829 * if (err != TS_SUCCESS) { *
830 * // fortunately, 'in' has not been changed *
831 * } *
832 * *
833 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
834 * changed allowing you to handle the error properly. *
835 * *
836 * Unless stated otherwise, the order of the parameters for transformation *
837 * functions is: *
838 * *
839 * function(input, [additional_input], output, [additional_output]) *
840 * *
841 * 'additional_input' are parameters required to calculate the actual *
842 * transformation. 'additional_output' are parameters storing further result. *
843 * *
844 * Note: None of TinySpline's transformation functions frees the memory of the *
845 * output parameter. Thus, when using the same output parameter multiple *
846 * times, make sure to free memory before each call. Otherwise, you will *
847 * have a bad time with memory leaks: *
848 * *
849 * tsBSpline in = ... // an arbitrary spline *
850 * tsBSpline out; // result of transformations *
851 * *
852 * ts_bspline_to_beziers(&in, &out); // first transformation *
853 * ... // some code *
854 * ts_bspline_free(&out); // avoid memory leak. *
855 * ts_bspline_buckle(&in, &out); // next transformation *
856 * *
857 * If you want to modify your input directly without having a separate output, *
858 * pass it as input and output at once: *
859 * *
860 * tsBSpline s = ... // an arbitrary spline *
861 * tsReal *knots = ... // a knot vector *
862 * *
863 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
864 * *
865 * Note: If a transformation function fails *and* input != output, all fields *
866 * of the output parameter are set to 0/NULL. If input == output, your *
867 * input may have an invalid state in case of errors. *
868 * *
869 ******************************************************************************/
