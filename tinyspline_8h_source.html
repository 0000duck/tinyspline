830 /******************************************************************************
831 * *
832 * :: Transformation functions *
833 * *
834 * TinySpline is a library focusing on transformations. That is, most *
835 * functions are used to transform splines by modifying their state, e.g., *
836 * their number of control points, their degree, and so on. Accordingly, each *
837 * transformation functions specifies an input and output parameter (along *
838 * with the other parameters required to calculate the actual transformation). *
839 * By passing a different pointer to the output parameter, the transformation *
840 * result is calculated and stored without changing the state of the input. *
841 * This is in particular useful when dealing with errors as the original state *
842 * will never be modified. For instance, let's have a look at the following *
843 * code snippet: *
844 * *
845 * tsBSpline in = ... // an arbitrary spline *
846 * tsBSpline out; // result of transformation *
847 * *
848 * // Subdivide 'in' into sequence of bezier curves and store the result *
849 * // in 'out'. Does not change 'in' in any way. *
850 * tsError err = ts_bspline_to_beziers(&in, &out); *
851 * if (err != TS_SUCCESS) { *
852 * // fortunately, 'in' has not been changed *
853 * } *
854 * *
855 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
856 * changed allowing you to handle the error properly. *
857 * *
858 * Unless stated otherwise, the order of the parameters for transformation *
859 * functions is: *
860 * *
861 * function(input, [additional_input], output, [additional_output]) *
862 * *
863 * 'additional_input' are parameters required to calculate the actual *
864 * transformation. 'additional_output' are parameters storing further result. *
865 * *
866 * Note: None of TinySpline's transformation functions frees the memory of the *
867 * output parameter. Thus, when using the same output parameter multiple *
868 * times, make sure to free memory before each call. Otherwise, you will *
869 * have a bad time with memory leaks: *
870 * *
871 * tsBSpline in = ... // an arbitrary spline *
872 * tsBSpline out; // result of transformations *
873 * *
874 * ts_bspline_to_beziers(&in, &out); // first transformation *
875 * ... // some code *
876 * ts_bspline_free(&out); // avoid memory leak. *
877 * ts_bspline_buckle(&in, &out); // next transformation *
878 * *
879 * If you want to modify your input directly without having a separate output, *
880 * pass it as input and output at once: *
881 * *
882 * tsBSpline s = ... // an arbitrary spline *
883 * tsReal *knots = ... // a knot vector *
884 * *
885 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
886 * *
887 * Note: If a transformation function fails *and* input != output, all fields *
888 * of the output parameter are set to 0/NULL. If input == output, your *
889 * input may have an invalid state in case of errors. *
890 * *
891 ******************************************************************************/
