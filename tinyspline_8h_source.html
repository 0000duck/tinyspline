504 /******************************************************************************
505 * *
506 * Transformation functions *
507 * *
508 * TinySpline is a library focusing on transformations. That is, most *
509 * functions are used to transform splines by modifying their state, e.g., *
510 * their number of control points, their degree, and so on. Accordingly, each *
511 * transformation functions specifies an input and output parameter (along *
512 * with the other parameters required to calculate the actual transformation). *
513 * By passing a different pointer to the output parameter, the transformation *
514 * result is calculated and stored without changing the state of the input. *
515 * This is in particular useful when dealing with errors as the original state *
516 * will never be modified. For instance, let's have a look at the following *
517 * code snippet: *
518 * *
519 * tsBSpline in = ... // an arbitrary spline *
520 * tsBSpline out; // result of transformation *
521 * *
522 * // Subdivide 'in' into sequence of bezier curves and store the result *
523 * // in 'out'. Does not change 'in' in any way. *
524 * tsError err = ts_bspline_to_beziers(&in, &out); *
525 * if (err != TS_SUCCESS) { *
526 * // fortunately, 'in' has not been changed *
527 * } *
528 * *
529 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
530 * changed allowing you to handle the error properly. *
531 * *
532 * Unless stated otherwise, the order of the parameters for transformation *
533 * functions is: *
534 * *
535 * function(input, [additional_input], output, [additional_output]) *
536 * *
537 * 'additional_input' are parameters required to calculate the actual *
538 * transformation. 'additional_output' are parameters storing further result. *
539 * *
540 * Note: None of TinySpline's transformation functions frees the memory of the *
541 * output parameter. Thus, when using the same output parameter multiple *
542 * times, make sure to free memory before each call. Otherwise, you will *
543 * have a bad time with memory leaks: *
544 * *
545 * tsBSpline in = ... // an arbitrary spline *
546 * tsBSpline out; // result of transformations *
547 * *
548 * ts_bspline_to_beziers(&in, &out); // first transformation *
549 * ... // some code *
550 * ts_bspline_free(&out); // avoid memory leak. *
551 * ts_bspline_buckle(&in, &out); // next transformation *
552 * *
553 * If you want to modify your input directly without having a separate output, *
554 * pass it as input and output at once: *
555 * *
556 * tsBSpline s = ... // an arbitrary spline *
557 * tsReal *knots = ... // a knot vector *
558 * *
559 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
560 * *
561 * Note: If a transformation function fails *and* input != output, all fields *
562 * of the output parameter are set to 0/NULL. If input == output, your *
563 * input may have an invalid state in case of errors. *
564 * *
565 ******************************************************************************/
