883 /******************************************************************************
884 * *
885 * :: Transformation functions *
886 * *
887 * TinySpline is a library focusing on transformations. That is, most *
888 * functions are used to transform splines by modifying their state, e.g., *
889 * their number of control points, their degree, and so on. Accordingly, each *
890 * transformation functions specifies an input and output parameter (along *
891 * with the other parameters required to calculate the actual transformation). *
892 * By passing a different pointer to the output parameter, the transformation *
893 * result is calculated and stored without changing the state of the input. *
894 * This is in particular useful when dealing with errors as the original state *
895 * will never be modified. For instance, let's have a look at the following *
896 * code snippet: *
897 * *
898 * tsBSpline in = ... // an arbitrary spline *
899 * tsBSpline out; // result of transformation *
900 * *
901 * // Subdivide 'in' into sequence of bezier curves and store the result *
902 * // in 'out'. Does not change 'in' in any way. *
903 * tsError err = ts_bspline_to_beziers(&in, &out); *
904 * if (err != TS_SUCCESS) { *
905 * // fortunately, 'in' has not been changed *
906 * } *
907 * *
908 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
909 * changed allowing you to handle the error properly. *
910 * *
911 * Unless stated otherwise, the order of the parameters for transformation *
912 * functions is: *
913 * *
914 * function(input, [additional_input], output, [additional_output]) *
915 * *
916 * 'additional_input' are parameters required to calculate the actual *
917 * transformation. 'additional_output' are parameters storing further result. *
918 * *
919 * Note: None of TinySpline's transformation functions frees the memory of the *
920 * output parameter. Thus, when using the same output parameter multiple *
921 * times, make sure to free memory before each call. Otherwise, you will *
922 * have a bad time with memory leaks: *
923 * *
924 * tsBSpline in = ... // an arbitrary spline *
925 * tsBSpline out; // result of transformations *
926 * *
927 * ts_bspline_to_beziers(&in, &out); // first transformation *
928 * ... // some code *
929 * ts_bspline_free(&out); // avoid memory leak. *
930 * ts_bspline_buckle(&in, &out); // next transformation *
931 * *
932 * If you want to modify your input directly without having a separate output, *
933 * pass it as input and output at once: *
934 * *
935 * tsBSpline s = ... // an arbitrary spline *
936 * tsReal *knots = ... // a knot vector *
937 * *
938 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
939 * *
940 * Note: If a transformation function fails *and* input != output, all fields *
941 * of the output parameter are set to 0/NULL. If input == output, your *
942 * input may have an invalid state in case of errors. *
943 * *
944 ******************************************************************************/
